基础知识

三层架构：

  • controller：控制层，接收前端发送请求并处理、响应数据
  • service：业务逻辑层，处理具体业务逻辑
  • dao(Data Access Object)：持久层，负责数据的增删改查

控制反转(IOC，Inversion Of Control)：对象的创建控制权由程序自身转移到容器

依赖注入(DI，Dependency Injection)：容器为应用程序提供运行时所依赖的资源。

Bean对象：IOC容器中创建、管理的对象。

在项目开发时，其中service层和dao层应先定义对应的接口，再实现实现类。

IOC

@Component将该实现类产生的对象交给容器。

@Autowired运行时自动查找该类型的Bean对象并赋值给该成员变量。

@Controller、@Service、@Repository为@Component的衍生注释，分别对应controller、service、dao层对象。

Bean对象的名称默认为类名（首字母小写）。

声明Bean的注解要想生效，需要被组件扫描注解@ComponentScan扫描。

主程序@SpringBootApplication中包含了@ComponentScan注解，能扫描启动类所在包极其子包内的所有@Component。

DI

@Autowired进行依赖注入的方式：

  • 属性注入：

  • 构造函数注入：如果当前类只有一个有参构造函数，则不需要声明@Autowired注解

  • setter注入

@Autowired注入是通过类型进行注入的，若存在多个同类型Bean，会报错。

解决方案：

  • @Primary：在需要的Bean对象上添加注解@Primary
  • @Qualifier：在注入的位置，结合@Autowired，额外添加注解@Qualifier("beanName")，此处的beanName遵循Bean对象的命名规范
  • @Resource：将@Autowired替换为@Resource(name="beanName")。

Spring中的事务提交

使用注解@Transactional，将当前方法交给spring进行事务管理。添加在业务层的方法、类、接口上。实际项目中建议加在对数据进行多次增删改的方法上。

对springboot配置文件修改，设置允许事务管理的输出级别为debug：

rollbackFor

@Transactional默认出现运行时异常RuntimeException才会回滚，可以通过设置参数rollbackFor控制出现何种异常时执行rollback操作。例如出现所有异常都回滚，则指定@Transactional(rollbackFor={Exception.class})。

propagation

事务传播行为：指的就是当一个事务方法被另一个事务方法调用时，这个事务方法应该如何进行事务控制。

可以通过propagation参数指定，常用的属性为：

  • REQUIRED：[默认值]，需要事务，有则加入，无则创建新事务。

  • REQUIRES_NEW：需要新事务，无论有无，总是创建新事务。

应用场景：在新增数据时，无论是否成功，都要记录操作日志。默认情况下使用try finally语句，产生异常时事务管理最终也会对记录日志的操作进行回滚，不符合需求。因此需要对日志记录开启一个新的事务，即对日志记录的service层的方法指定注解@Transactional(propagation = Propagation.REQUIRES_NEW)。

登陆校验

会话技术：

  • 客户端会话跟踪技术：Cookie。
    • http协议支持，响应头：Set-Cookie，请求头：Cookie。
    • 移动端APP不能使用。Cookie存放在浏览器本地，用户可随意修改与禁用，不安全。同时Cookie不能跨域（协议、IP/域名、端口不一致就涉及到跨域）。

  • 服务端会话跟踪技术：Session

    • 底层基于Cookie实现，存储在服务端，安全。
    • 在服务器集群环境下无法使用Session，也包含Cookie的缺点。

  • 令牌技术（主流方案）

    • 支持PC端和移动端
    • 解决了集群环境下的认证问题
    • 减轻服务器端存储压力

JWT令牌

JSON Web Token，用于通信双方以json数据格式安全地传输信息。

组成：

  • Header：记录令牌类型、签名算法等。如：{"alg":"HS256", "type":"JWT}。
  • Payload：携带一些自定义信息与默认信息，如：{"id":"1","username":"Tom"}。
  • Signature：防止Token被篡改，将header与payload融入并加入指定密钥，通过指定签名算法计算而来。

JWT校验时的密钥与生成时的密钥应一致。当令牌过期或被篡改会报错。

过滤器Filter

过滤器可以把对资源的请求拦截下来，完成一些通用的操作：如登陆校验、统一编码处理、敏感字符处理等。

步骤：

  • 定义Filter：定义一个类实现Filter接口以及方法（init、doFilter、destroy），一般只需要实现doFilter。
  • 配置Filter：Filter类上加上@WebFilter注解，配置拦截路径urlPatterns。引导类上加@ServletComponentScan开启Servlet组件支持。

Bean原理

Bean作用域

  • singleton：容器内同名称的bean只有一个实例（单例，默认，会在项目启动时创建，创建完毕后直接存入IOC容器，加上@Lazy会在第一次使用时创建bean对象）
  • prototype：每次使用该bean时会创建新的实例（多例，通过注解@Scope("prototype")指定）

不保存数据的对象即无状态bean对象，可以使用单例，否则使用多例。

第三方Bean

要管理第三方Bean，需要使用@Bean注解。可以定义一个配置类对第三方Bean集中管理。

如果第三方Bean需要依赖其他Bean对象，直接在Bean定义方法中设置形参，容器会根据类型自动装配。

@Bean可以通过name或value设置bean的名称，不指定则为方法名。

自动配置

springboot启动后，一些配置类、bean对象就自动存入了IOC容器中。

导入

springboot启动类默认只能扫描启动类所在包及其子包。当使用第三方包时可以通过：

  • 增加注解@ComponentScan(basePackages={"com.example"})指定扫描位置，注意此时默认位置不起作用，需要再手动指定。该方式使用繁琐，且性能并不高。

  • 使用@Import注解导入：

    • 导入普通类

    • 导入配置类

    • 导入ImportSelector的实现类

    • 第三方一般会将ImportSelector封装在@Enablexxx的注解上，此时只需要将该注解加载启动类上。

@Conditional注解

@Conditional：满足一定条件才会注册bean对象到IOC容器中，可以作用在方法和类上。常见的有：

  • @ConditionalOnClass：当环境中有对应字节码文件时，才注册。

  • @ConditionalOnMissingBean：当环境中没有对应的bean时，才注册。

  • @ConditionalOnProperty：配置文件中有对应的属性和值才注册。

    @ConditionalOnProperty(name = "key", value = "test")当配置文件中key键的值为test时才会创建bean对象。

SpringCache

实现了基于注解的缓存功能。SpringCache提供了一层抽象，底层可以切换如EHCache、Caffeine、Redis等缓存实现。

  • @EnableCaching：开启缓存注解功能，通常加在启动类上

  • @Cacheable：在方法执行前先查询缓存中是否有数据，如果有数据，则直接返回；否则调用方法并将方法返回值放到缓存中

  • @CachePut：在执行方法后，将返回值放到缓存中
